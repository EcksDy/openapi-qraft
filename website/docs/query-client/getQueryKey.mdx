# getQueryKey(...)

The method provides a standardized way to generate `QueryKey` for _Queries_.
See TanStack [_Query Keys ðŸŒ´_](https://tanstack.com/query/latest/docs/framework/react/guides/query-keys) guide for more information.

```ts
const queryKey = qraft.<service>.<operation>.getQueryKey(parameters);
```

### Arguments

1. - `parameters: { path, query, header } | QueryKey | {}`
     - **Required**, OpenAPI request parameters for the query, strictly-typed âœ¨
     - `parameters` will be used to generate the `QueryKey`
     - If operation does not require parameters, you must pass an empty object `{}` for strictness

### Returns

`QueryKey` - a query key for the _Queries_

### Example

```tsx
const queryKey = qraft.files.getFiles.getQueryKey({
  header: { 'x-monite-version': '1.0.0' },
  query: { id__in: ['1', '2'] },
});

// `queryKey` will be an array of objects

expect(queryKey).toEqual([
  { method: 'get', url: '/files' },
  {
    header: { 'x-monite-version': '1.0.0' },
    query: { id__in: ['1', '2'] }
  }
]);
```
